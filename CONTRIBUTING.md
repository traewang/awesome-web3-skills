# Contributing to Awesome Web3 Skills

Thank you for your interest in contributing! This list is curated to maintain high quality, so please review the guidelines below before submitting a PR.

## Guidelines

- **One resource per pull request.** This makes it easier to review and discuss.
- **Use the following format:** `- [Name](url) - Brief description ending with a period.`
- **Add new entries at the bottom** of the relevant category.
- **Descriptions should be concise** - one sentence explaining what it is and why it's useful.
- **Check for duplicates** before submitting.
- **Verify that the link works** and the resource is actively maintained.

## Quality Criteria

Resources should meet the following standards:

- **Actively maintained** - Last updated within the past 12 months (for tools/projects).
- **Well-documented** - Has clear documentation or README.
- **Community adoption** - Has meaningful usage or community support.
- **Relevant** - Directly related to Web3 development skills.

## How to Submit

1. Fork this repository.
2. Create a new branch: `git checkout -b add-resource-name`.
3. Add your resource to the appropriate section in `README.md`.
4. Commit your changes: `git commit -m "Add [resource name]"`.
5. Push to your fork: `git push origin add-resource-name`.
6. Open a Pull Request with a brief explanation of why this resource should be included.

## Suggesting New Categories

If you believe a new category is needed, please open an issue first to discuss it before submitting a PR.

## Code of Conduct

Be respectful and constructive. We are all here to help the Web3 developer community grow.
